## Java キーワード検索 Excel とは？
Java キーワード検索 Excel とは、Java コードを使用して Excel ワークシート内の特定の単語やフレーズをプログラム的にスキャンするプロセスを指します。GroupDocs.Parser を使用すれば、Excel を開かずにセルの値を読み取り、迅速なキーワードマッチを実行し、各出現箇所の正確な位置（シート、行、列）を取得できます。

## テキスト検索に GroupDocs.Parser を使用する理由
- **速度とメモリ効率:** 完全な Office アプリケーションをロードせずにドキュメントを解析します。  
- **統一された API:** 同じコードで Excel、PDF、Word、PowerPoint、HTML、EPUB、メール形式を処理できます。  
- **高度な機能:** 正規表現サポート、ページ単位の検索、結果のハイライトが可能です。  
- **クロスプラットフォーム:** デスクトップアプリからクラウドサービスまで、Java 対応環境ならどこでも動作します。

## よくある質問

**Q:** 商用プロジェクトで GroupDocs.Parser for Java を使用できますか？  
**A:** はい、商用ライセンスを取得すれば、ライブラリを任意の本番アプリケーションに統合できます。

**Q:** ライブラリはパスワード保護された Excel ファイルをサポートしていますか？  
**A:** もちろんです。ワークブックを開く際にパスワードを渡せば、パーサーがリアルタイムで復号します。

**Q:** 大規模なスプレッドシートでキーワード検索のパフォーマンスはどのようにスケールしますか？  
**A:** GroupDocs.Parser はストリーミング方式でデータを読み込むため、数千行のファイルでもメモリ使用量は低く抑えられます。非常に大きなファイルの場合は、シート単位で処理することを検討してください。

**Q:** 同一クエリでキーワード検索と正規表現を組み合わせられますか？  
**A:** はい。まずシンプルなキーワードフィルタを実行し、次にマッチしたセルに対して正規表現パターンを適用して細かく制御できます。

**Q:** サンプルプロジェクトや Maven の座標はどこで見つけられますか？  
**A:** 公式ドキュメントと GitHub リポジトリに、すぐに実行できるサンプルと正確な Maven/Gradle 依存スニペットが含まれています。
